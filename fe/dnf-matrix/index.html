DNF LoRA — Cross-base Inference Matrix

5 LoRA × 6 Base × 5 Prompt = 150 images · 다른 base 에 학습한 LoRA 들이 다른 base 에서 어떻게 작동하는가

실험 개요

DNF 공식 일러스트 522장으로 5개의 다른 base 모델에 LoRA 를 각각 학습하고, 학습된 5 LoRA 를 6개의 base 모델(학습 5종 + V-Pred 1종)에 끼워 넣어 추론 결과를 비교. 같은 계통 (family tree) 내에서 cross-base inference 가 작동하는지, 어떤 경계에서 깨지는지 정량 확인.

SDXL 계열 모델 계통도

SDXL 1.0 base ← stabilityai/stable-diffusion-xl-base-1.0 ├── Animagine XL 3.1 (SDXL 직계 fine-tune) └── Illustrious XL v0.1 (SDXL 직계 fine-tune, Danbooru 학습) ├── NoobAI-XL v1.1 epsilon (Illustrious 파생) │ └── NoobAI-XL V-Pred v1 (같은 v1.1 에서 prediction_type 재학습) └── WAI-Illustrious v16 (Illustrious 파생)

5 LoRA — 각각 다른 base 에 학습

같은 데이터셋(522장 + ssw_dnf 트리거 + 동일 캡션) · 같은 하이퍼파라미터(rank 32, α 16, LR 1e-4, 3000 step, AdamW, bf16) · seed 42 · 학습은 RTX 5070 Ti (16GB) 에서 각 4.5시간, 총 22시간.

핵심 발견

  • NoobAI V-Pred 열 100% 실패: epsilon 으로 학습한 LoRA 5개 전부 v_prediction base 에서 검은색/노이즈로 깨짐. prediction_type 변환은 LoRA 호환의 절대 경계.
  • Illustrious 트리 내 cross-base 호환 양호: Illustrious / NoobAI eps / WAI 끼리 LoRA 교체해도 자연스러운 스타일 유지. 운영 효율 관점에서 매우 유용.
  • Animagine 의 관대함: 어떤 LoRA 를 적용해도 받아들임. SDXL 직계 fine-tune 의 특성으로 추정.
  • 계통 경계는 부분 호환: SDXL 직계 ↔ Illustrious 트리 — 그림은 나오지만 스타일이 변함. 두 계통이 SDXL 1.0 공통 조상이라 부분 호환되는 것으로 해석.
  • Metric 의 한계: CLIP/DINO 단독으로는 V-Pred 깨짐을 약하게만 잡음 (CLIP Δ 0.011, DINO 는 오히려 V-Pred 가 살짝 높음). 정성+정량 두 트랙 병행 필요.
